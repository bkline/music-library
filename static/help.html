Getting Started

To use this application, you will need a browser with JavaScript enabled (if you don't know what that means, it's enabled) a connection to the internet, and an active account for the application. The software is mobile-friendly, so you can use it from a desktop computer, a laptop, a tablet, or a smart phone.

To log into the catalog application, connect to its address using your browser, enter the account name and password, and click the Login button. You will see a page which you can use to browse the music library catalog (see Browse below), with a menu for the different sections of the application, including a button for logging out when you are done. If you have not been using the application for a couple of days your session will be logged out and you will be asked to log back in to resume working with the catalog.

As you work with the application, it will communicate with alert boxes which tell you when an action you have performed has succeeded, or if problems were encountered (such as validation errors on a form) what those problems are. These notifications will slide off after several seconds (though you can select them to keep them visible longer) or you can dismiss them immediately using the X in the upper right corner of the box.

Browsing

The Browse Catalog page lets you scroll through the list of works contained in the music library. You can view the complete catalog, or you can customize the display using various combinations of filtering fields, effectively providing you with a powerful reporting tool. These reports can be used to answer such questions as Which works have not been performed in the past three years? or Which items have we added to the library this year? or Which pieces are out on loan (and to whom)? Reports can be generated as web HTML, displayed in your browser, or as an Excel spreadsheet. The report request form is grouped into four blocks of fields, each of which is collapsed when the form first appears. To open a block of fields, click on its label. To close an open block, click the label bar a second time. When your request is ready, click Report at the bottom of the form and your report will be displayed in a newly opened browser tab, or (if you chose to have an Excel report created) downloaded to your computer or mobile device. You can click the Reset button whenever you want to restore all the fields on the form to their default values. If you just want to scroll through the entire catalog without any filtering, you can simply click the Report button directly, without specifying any filtering options.

The Options block has fields for the title to be displayed at the top of the report, for the report's format (web page or Excel spreadsheet), how the report should be sorted, and which columns should be displayed on the report. The Report Title and Report Columns fields are the only ones which are required (though they both have reasonable defaults). To add another column to the report, click on the Report Columns field and select the name of the desired column. This is a "typeahead" field, so you can narrow the list of displayed options by typing part of the name of the column you want to add. To control the position of the column on the report after it has been selected for inclusion on the report, drag its name to change where it will appear. The order of the report is determined by the values in the first column. When two or more rows on the report have the same value for the first column, those rows are ordered based on the values in the second column, and so forth. The Report Sorting field in this block has an option for specifying that the order of the report should be reversed (for example, to get the most recently-performed pieces at the top of the report). For the most flexibility in reporting power, chose the Excel version of a report so you can take advantage of Excel's filtering, transformation, and sorting options. Excel is also a good option for reports which have many columns, which can be unwieldy in a web table.

The Identification block of the form has fields for filtering the report based on the works' title(s), creators (composer, arranger, lyricist), whether they appear in a given collection, and what ensemble they have been arranged for. For the Arrangement field, you can specify more than one value, and works which have been arranged for any of those ensembles will be selected. Each of these fields has support for controlling the logic for that field's filtering using radio buttons. For example, you can say you want only those works whose titles contain the value you enter, or you can ask to exclude the works whose titles contain the value you type.

The Classification block has fields for filtering the report based on keywords and/or tags which have been assigned to the works, as well as the occasions (sometimes referred to as "seasons") for which the work is intended to be performed (for example, Hanukkah or Easter or Wedding). Keywords are general terms (for example, "Beauty" or "Justice") which can be assigned to the works to assist in selecting pieces to fit a general theme. By contrast, category tags are grouped for specific purposes. For example, some of the tags are used to identify the performing ensemble for which the work was acquired ("Adult Choir" or "Handbells" for example). There are also tag groups for Faith Traditions, Languages, Periods, and Styles. As with the Arrangement field in the previous block, more than one value can be selected for a given field, and works which match any of those values will be included on the report. However, if values are selected for two or more separate fields, only those works which match at least one of the values in each of those fields will be included on the report.

The Activity block contains fields which you can use to select works based on what we have done with them. For example, you can narrow the report to works which were last performed no later than (or no earlier than) a specified date, or which were added to the catalog no later than (earlier than) a given date, or which are (or are not) out on loan.

If the Item ID column is included in the report (as it is by default) the value in that column will be wrapped in a link which can be used to open a new browser tab showing the printable version of the full record. When this version is actually printed (or saved as a PDF file, if your browser supports that as an alternative to printing) the banner and menu bar are suppressed so that they do not take up space in the printed output.

Editing

When you want to add a record to the catalog or modify an existing record, select Edit from the menu. At the top of the page which appears is a form you can use to find the record you want to modify (if that's what you're about to do) or to confirm that the record doesn't already exist if you're planning on adding it (always a good thing to check, so we don't end up with duplicate records, although different arrangements of the same musical composition should usually get their own records in the catalog). To search for a particular record in the catalog, enter at least three characters in the Title field or the Composer Or Arranger field (or specify values in both fields for further narrowing the search) and click the Search button. You will see a selection of the records which contain what you typed in the field(s) in the corresponding portion of those records. For example, if you enter flower in the Title field and click the Filter button, you will see two records in the list at the bottom of the page (at the time of this writing): Cantan Las Flores and I Got Me Flowers. Even though "flower" does not appear in the title of the first work, it matches because its alternate title is The Flowers Are Singing. For another example, if you enter sheep in the Title field, and bach in the Composer Or Arranger field, Bach's Sheep May Safely Graze will be found. If there are more than ten records which match the filtering criteria, a row of controls will appear underneath the list for navigating through the pages of matching records.

Each work listed has buttons on the right side of the page, appearing as icons representing their functions. The first button is a print button which can be used to bring up the full record in a separate browser tab, as described in the Browsing section. You can use your browser's Print command to send the record to the printer (or a PDF file, if your browser supports that feature).

The second button brings up the form for editing the record. If you hover your mouse above the title at the top of the form for a couple of seconds, you will see a small popup which displays the item number for the record you are editing. Normally, you will never need to know this number, but if you ever need to report a problem with a particular record to the development team, that number can be a useful piece of information.

The editing form is organized in multiple groups of fields. The first four groups are for Identification of the work being cataloged (titles, keywords, etc.), Creation (composer, arranger, lyricist), Publication (publisher, supplier, copyright information, and stock number), and Musical Information (key, time signature, accompaniment, arrangement, level of difficulty, etc.). The other four groups are for repeatable blocks of information: Performances (when performed, comments about the performance), Inventories (how many copies, when the inventory was taken, the location and condition of the copies, etc.), Parts (for works with orchestral accompaniment), and Loans (when and to whom was the work loaned, when was it returned, etc.).

When the form is first displayed, all of the eight groups are collapsed to make it easier to navigate to the part(s) of the form with which you need to work. To open a section, click on its title. To collapse an opened section, click on its title again. In the repeatable groups, each of the instances is contained in a nested collapsed set of fields, which can be opened and closed in the same way as the enclosing group. Each of the repeating groups has a button for adding a new instance, and each instance has a button at the far right of its title bar for removing that instance.

The fields on the editing form are of different types, most of which will be familiar, such as plain text fields (for example, the title fields), dropdown picklists (from which you select a value from a set of valid values), date fields (which provide an interface which makes it easy to select a date), or a checkbox (for indicating that the item being cataloged is actually a collection of individual works, each of which can get its own catalog record). For the person (composer, arranger, lyricist) and company (publisher, supplier) picklists, you can start typing and the selections shown in the dropdown list will be restricted to those which match what you have typed so far, even if what you typed appears in the middle of a value instead of at the beginning, making it easier to find specific values in these very long lists, which contain hundreds of values. If you discover that the person or company which you need to select for a work is not available on the picklist you can add the person or company without leaving your editing form by clicking on the Add Person or Add Company button at the bottom of the page. A popup overlays the editing form. Fill in at least the required fields, click the Add Record button, and the new value will appear in your picklist in the main editing form.

When you have finished editing the record for the work, click the Save button at the bottom of the page (or Cancel if you want to abandon any editing you have done and return to the Catalog Search page). If there are any validation errors found on the form when you try to save it, correct them and click the Save button again.

The final button on each line of the list of records in the Search results is for deleting the record. That button will be absent if the record is for a collection which is referenced by other records representing works contained in that collection, because you cannot delete such a record while any other records depend on it.

To create a new record, click the Create button underneath the search form. The form for creating a new record is the same form used for editing an existing record.

Lookup Values

All of the sets of values used for the picklists on the main editing form can be edited to add new values, as well as edit or remove existing values. In many cases, the order in which each value appears on its picklist can also be controlled. For accounts with permission to edit the catalog, a menu item labeled Manage Lookup Tables will be present. Clicking on that menu item opens a submenu from which you can select which value set you wish to edit. A form with a single picklist is then displayed, containing the names of the values currently present in the set. For those lookup sets which support overriding the default sort order which would result from alphabetical ordering, the sort position is also shown for each value on the list. Select an existing value to edit and click the Edit button, or click the Create button if you need to add a value which is not already in this set.

Perform the editing of the value's fields and click the Save button. If any validation errors are found, correct them and try again. For values which are used by the catalog's main item records, a count of how many records use this value is displayed at the top of the editing form for the value. Such values cannot be deleted from the list unless all of those uses are removed from the catalog records. If a value has no records which use it, a Delete button will appear below the editing form, so you can remove the value from the list if you believe it will no longer be needed.

The Tag Groups set of values is not used directly by the main catalog records, but is instead used to associate the Tag values to a specific category group. These categories can also be edited or added, though as with the other value sets, a tag group cannot be removed while any Tag values are members of it.

Accounts

The Accounts page lists all of the user login accounts in the system, showing the account name, the user's full name, the password, the user's role (editor, viewer, or administrator), whether the account is still active or has been retired, and a comment describing the account. The row for each account is clickable, bringing up a form for editing the account's information. There is also a Create Account button above the list of existing accounts for adding new users to the system. When creating a new account, be sure to uncheck the *Read Only* checkbox if the user of the account should be able to edit the catalog records.

The account records are the one type of record which cannot be removed, because that would lose information for the audit trail. If you find that you have created an account by mistake, and the account has never been used, mark the account as inactive, with a note explaining what happened, and then re-use that account record the next time a new user needs to be added to the system. But first look at the Audit Trail (see the next section of this manual) to make sure that the account really has never been used before recycling it.

Audit Trail

The Audit Trail page shows the history of all user activity resulting in modifications to the catalog data. This is occasionally useful when a user reports that the software is not behaving as expected. The system doesn't capture all the information needed to reconstruct each version of each record, but it does capture the basic information about each addition, modification, or deletion (who, what, when, which record, which table). The events are displayed in reverse chronological order (that is, newest activity at the top).
